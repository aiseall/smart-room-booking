# Smart Room Booking Constitution

## Core Principles

### I. Simple Stack
The project MUST use Python 3.12 + FastAPI for the backend and React 18 + TypeScript for the frontend. No additional backend frameworks or frontend meta-frameworks are permitted.

### II. Database Simplicity
MVP MUST use SQLite with aiosqlite for async support. No ORM abstraction leaks — all queries MUST go through SQLAlchemy async session. Migration to PostgreSQL SHOULD be possible by changing only the connection string.

### III. Test-First
Backend MUST use pytest; frontend MUST use vitest. Overall test coverage MUST be >= 80%. Critical business logic (conflict detection, auto-release) MUST have >= 90% coverage.

### IV. RESTful API with OpenAPI
All endpoints MUST be under `/api/v1`. OpenAPI 3.0 documentation MUST be auto-generated by FastAPI. All request/response schemas MUST be defined with Pydantic v2.

### V. Security
Authentication MUST use JWT tokens. Authorization MUST use RBAC with three roles: employee, team_admin, facility_admin. All input MUST be validated with Pydantic. Passwords MUST be hashed with bcrypt.

### VI. Performance
API P95 response time MUST be < 300ms. Frontend initial page load MUST be < 1.5s. Database queries MUST use appropriate indexes.

### VII. Simplicity First (YAGNI)
MVP MUST NOT use microservices, message queues, Redis, or any distributed systems. Start simple, scale later.

### VIII. Containerization
The application MUST support Docker deployment. Target platform is Azure Container Apps. A multi-stage Dockerfile MUST produce an image < 300MB. A `/api/v1/health` endpoint MUST exist for container health checks.

### IX. CI/CD
All code MUST pass GitHub Actions lint + test + build before merge. Coverage gate at 80% is non-negotiable. Docker image build MUST be verified in CI.

### X. Internationalization
All user-facing strings MUST support Chinese (zh-CN) and English (en) from day one. The frontend MUST use i18next for translations.

## Governance

This Constitution supersedes all ad-hoc decisions. Any amendment requires:
1. Documentation of the change and rationale
2. Review by project lead
3. Update to this file with new version number

All code reviews MUST verify Constitution compliance.
